AI 기반 PMS (Project Management System) 기획 및 설계 문서
문서 개요
목적
본 문서는 AI 기반 프로젝트 관리 시스템(PMS)의 추가 요건을 바탕으로 한 기획 및 설계 문서입니다. 첨부된 두 개의 회의 녹취록("PMS 요건협의_260114_1.txt" 및 "PMS 요건협의_260114_2.txt")을 참고하여, PMS의 기능 확장, 구조 설계, AI 통합 등을 체계적으로 정리하였습니다. 이 시스템은 AI 프로젝트(예: 보험 심사 자동화)를 위한 관리 도구로, 전통적인 SI(시스템 통합)와 AI 요소를 결합하여 효율적인 프로젝트 추적, 협업, 교육 관리를 목표로 합니다.
배경 및 분석 접근
사용자의 쿼리를 바탕으로, 회의 내용을 다차원적 분석(MDA)과 혁신적 솔루션(IS)을 혼합하여 분석하였습니다. MDA를 통해 시간(프로젝트 단계별 진척), 공간(시스템 계층 구조), 추상(개념적 통합), 인과(요구사항과 결과의 연계), 계층(전체-서브 프로젝트-태스크) 차원을 가중치(예: 진척률 40%, 교육 관리 20%)로 평가하였습니다. 이를 IS와 결합하여 기존 PMS의 조합(Scrum 기반 + AI 오케스트레이션), 참신성(AI 자동 요구사항 분류), 실현 가능성(GPU 서버 활용), 가치(효율성 향상), 위험(버틀넥 발생 시 온/오프 스위치)을 고려한 혁신적 설계를 도출하였습니다. 결과적으로, PMS를 하이어라키 구조로 재정의하여 유연성과 확장성을 강조합니다. (분석 과정: 회의 내용을 시간 차원으로 분해 → 인과적으로 요구사항 추적 → 계층적으로 구조화 → IS를 적용해 솔루션 제안.)
대상 사용자

PMO(프로젝트 관리 사무소) 담당자: 전체 대시보드 및 통계 활용.
파트 리더(예: AI 개발, SI 개발): 단계별 관리.
팀원: 태스크 및 이슈 추적.
고객: 요구사항 검수 및 교육 히스토리 확인.

시스템 요구사항 요약

기능적 요구: 대시보드, 단계별 관리, 공통 관리(산출물, 회의, 이슈), 교육 관리(IT/현업), 채팅 기능.
비기능적 요구: AI 통합(LLM 활용), GPU 서버 지원(20-30명 규모), 성능(버틀넥 최소화), 보안(히스토리 관리).
제약: 개발 서버 필요(4090 GPU 4장 이상), 클라우드 기반 이벤트 트리거.

                                      → 교육 관리

기능 명세
1. 통합 대시보드

설명: PM/사업 관리자가 전체 프로젝트를 한눈에 보는 화면. 단계별 진척률, 통계(바 차트, 파이 차트), 요구사항 상태(완료/진행중/미착수).
상세 기능:
진척률: 전체 요구사항 100개 중 AI 30개(완료 70%), SI 30개(완료 50%) 등.
통계: 주간/월간 바 형태 그래프, 색상 마킹(녹색: 완료, 황색: 진행, 적색: 미착수).
필터: 파트 리더별, 기간별.

AI 통합: LLM으로 자동 요약(예: "이번 주 지연 이슈 3건").
가중치(MDA 분석): 시간 차원(실시간 업데이트, 40% 영향), 계층 차원(전체 → 서브 프로젝트).

2. 단계별 관리

설명: AI 프로젝트 스텝(요구사항 분석 → 데이터 수집 → OCR 처리 → AI 적용 → 테스트 → 운영 → 검증)과 SI 스텝(분석 → 설계 → 개발 → 테스트)을 투 트랙으로 관리. 스프린트 기반(Scrum).
상세 기능:
서브 프로젝트: AI 챗봇 개발, 모바일 앱 리뉴얼 등. 각 서브에 스프린트(태스크 리스트, 스토리 포인트, 번다운 차트).
요구사항 관리: 엑셀 업로드/다운로드, ID 부여, 업데이트(진행률 수동/자동 수정).
투 트랙: AI 트랙(엔드 투 엔드 우선)과 SI 트랙 병행, 상호 연계(인과 차원: AI 결과 → SI 화면 설계).

혁신(IS): 조합(Scrum + AI 자동 태스크 생성), 참신성(LLM으로 요구사항 분류), 위험(버틀넥 시 롤백 기능).
UI: 왼쪽 메뉴(1단계 ~ N단계), 오른쪽 상세(통계 + 리스트).

3. 공통 관리

설명: 기능 외 공통 요소 관리. 산출물, 회의, 이슈 중심으로 최소화(필수 3가지).
상세 기능:
산출물 관리: 목록(제출일, 검수일, 버전), 히스토리, 검수 확인서 업로드.
회의 관리: 일정(주간/착수/종료 보고), 히스토리, 요구사항 녹아듦.
이슈 관리: 발생 담당자, 확인자, 마감일, 진행 상황 추적.

AI 통합: 챗 기능(내부 소통, 히스토리 저장, PostgreSQL + GraphDB 연동).
MDA 분석: 인과 차원(이슈 → 진척 영향, 30% 가중), 공간 차원(전체 프로젝트 영향).

4. 교육 관리 (교육 담당자용)

설명: IT 대상(개발자 수준)과 현업 대상(업무 수준)으로 분리. 사전/사후 교육 지원.
상세 기능:
IT 교육: 에이전트 AI, 머신러닝, 딥러닝, 파이썬 등. 역할별 로드맵(기초 → 중급 → 고급).
현업 교육: AI 인식, 사례 기반(기획/운영). 히스토리(참여자, 날짜, 세부 목차).
로드맵: 매트릭스(역할 × 레벨), 미니 프로젝트/캡스톤 포함.
사후 교육: 구축 에이전트 역할 설명(버전, 기능).

혁신(IS): 가치(교육을 통해 프로젝트 정보 발견), 실현 가능성(오픈 소스 의존), 위험(고객 레벨 맞춤).
UI: 메뉴 클릭 시 맵 표시, 클릭 → 세부 커리큘럼 + 히스토리.

5. AI 에이전트 오케스트레이션 통합 (옵션사항, 향후 개발)

설명: PMS와 연동(번들 패키지). 에이전트 팀원 개념(휴먼-에이전트 통제).
상세 기능:
모니터링: 큐/버틀넥, 대시보드 표시(모바일 지원).
트리거: 이벤트 기반(승인 → 다음 에이전트 자동 시작).
위험 대응: 온/오프 스위치, 롤백(엔드 투 엔드 성능 영향 최소화).

MDA 분석: 추상 차원(로직 플로우 vs. 비즈니스 플로우), 계층 차원(에이전트 레벨 오케스트레이션).
IS: 참신성(클라우드 풀 기반), 가치(리드 타임 감소), 위험(SI 업체 협업 시 분리 옵션).

UI/UX 설계

메인 레이아웃: 왼쪽 사이드바(통합 대시보드 > 단계별 관리(1~N) > 공통 관리(산출물/회의/이슈) > 교육 관리(IT/현업)).
흐름: 로그인 → 대시보드(통계) → 메뉴 클릭 → 상세(리스트 + 그래프) → 업데이트(퍼센트 수정) → 저장(히스토리).
가독성: 한눈에 보이는 통계, 색상 코딩, 모바일 최적화.
사용자 흐름 예시:
PM: 대시보드 → 단계별 진척 확인 → 이슈 등록.
팀원: 단계별 → 태스크 업데이트 → 채팅 소통.
고객: 요구사항 검수 → 교육 히스토리 조회.
