## 💬 3단계: AI 챗봇 열기

로그인 후 다음 중 하나를 클릭하세요:
- 우측 하단의 **AI 어시스턴트 아이콘** 🤖
- 상단 메뉴의 **AI 어시스턴트** 버튼

## 🎯 4단계: 테스트 시나리오

### 시나리오 1: 추천 질문 사용

1. AI 어시스턴트 패널에서 **"WBS 생성"** 버튼 클릭
2. 입력란에 자동으로 질문이 채워짐
3. **전송** 버튼 클릭 ✉️
4. 로딩 애니메이션 확인 (점 3개 튀어오름)
5. AI 응답 확인

### 시나리오 2: 직접 질문

**관리자/PM 계정:**
```
안녕하세요, 프로젝트 일정 관리를 도와줄 수 있나요?
```

**개발자 계정 (dev@insure.com):**
```
이번 스프린트에서 내가 해야 할 개발 작업을 알려줘
```

**QA 계정:**
```
테스트 케이스 작성을 도와줄 수 있나요?
```

**기대 결과:**
- 5-10초 내 AI 응답
- 한국어로 자연스러운 답변
- 타임스탬프 표시
- confidence: 0.85

### 시나리오 3: 연속 대화

첫 번째 질문:
```
현재 진행 중인 프로젝트를 알려줘
```

두 번째 질문:
```
그럼 다음 스프린트 계획은?
```

**확인사항:**
- 이전 대화 컨텍스트가 유지되는가?
- 자연스러운 대화 흐름인가?

## ✨ 성공 확인 사항

챗봇이 정상 동작하는 경우:

✅ **사용자 메시지**
- 우측에 파란색 말풍선으로 표시
- 타임스탬프 표시

✅ **AI 응답**
- 좌측에 회색 말풍선으로 표시
- "AI 분석" 레이블과 ✨ 아이콘
- 실제 프로젝트 관련 답변 (Mock이 아님!)

✅ **로딩 상태**
- 점 3개가 튀어오르는 애니메이션
- "AI 분석" 표시

## ❌ 문제 해결

### "통신할 수 없습니다" 오류

**해결:**
```bash
docker-compose restart backend llm-service
```

### Mock 응답이 나오는 경우

**증상:**
```
안녕하세요! PMS AI 어시스턴트입니다.
현재 Mock 모드로 동작 중입니다.
```

**해결:**
```bash
# LLM 서비스 상태 확인
curl http://localhost:8000/health

# LLM 서비스 재시작
docker-compose restart llm-service

# 30초 대기 후 다시 테스트
```

### 로그인 실패

**확인:**
- 이메일: `admin@insure.com` (@ 뒤가 insure.com인지 확인)
- 비밀번호: `admin123` (오타 확인)
