AI 기반 PMS 추가 기능 기획 및 설계서 1. 개요 1.1 목적 본 문서는 기존 AI 기반 프로젝트 관리 시스템(PMS)에 RFP(Request for Proposal) 관리 기능을 추가로 구현하기 위한 기획 및 설계서입니다. RFP 업로드를 통해 Graph RAG(Retrieval-Augmented Generation) 엔진을 활용하여 요구사항을 자동 추출하고 관리하며, 요구사항 ID 자동 부여, 스프린트 매핑, 진행률 업데이트, 프로젝트별 테넌트 관리, Graph RAG DB 테넌트 지원, 담당자별 AI 보고서 생성 기능(사용자가 버튼 누르면 생성), 생성된 보고서 버전관리 기능을 포함합니다. 추가로 RFP가 없더라도 요구사항에 별도 파일 업로드를 지원하여 유연한 요구사항 관리를 가능하게 합니다. 이 기능을 통해 PMS의 자동화 수준을 높이고, 프로젝트 관리의 효율성과 정확성을 강화하며, AI를 활용한 실시간 추적 및 보고 시스템을 구축합니다. 궁극적으로 프로젝트 지연을 최소화하고, 팀 협업을 촉진하며, 비즈니스 의사결정을 지원하는 통합 플랫폼을 실현합니다. 1.2 범위  RFP 업로드 및 Graph RAG 기반 요구사항 자동 추출/관리. 요구사항 ID 자동 부여, 스프린트 매핑, 진행률 및 상태(진행 중/완료) 업데이트. RFP 미존재 시 요구사항 별도 파일 업로드 지원. 프로젝트별 테넌트 관리 (PMS 전체 및 Graph RAG DB). 담당자별 AI 보고 생성 기능. 기존 PMS와의 통합: 사용자 인증, 데이터 공유, API 연동을 가정하며, 새로운 기능이 기존 워크플로우에 seamless하게融入되어 운영 비용을 절감하고 사용자 경험을 향상시킵니다.  2. 기능 요구사항 2.1 RFP 관리 기능 RFP 파일(문서, PDF 등)을 업로드하면 Graph RAG 엔진이 자동으로 내용을 분석하여 요구사항을 추출합니다. 추출된 요구사항은 수정 가능 해야함. 요구사항 수정시 Graph RAG도 업데이트. Graph RAG는 텍스트를 노드(요구사항)와 엣지(관계)로 그래프화하여 저장, 검색 효율을 높입니다. 프로젝트별로 RFP 및 요구사항이 관리되도록 테넌트 분리 되어야 함.  업로드 프로세스: 사용자 인터페이스(UI)에서 파일 드래그-앤-드롭 또는 선택 업로드. 업로드 후 백그라운드에서 Graph RAG가 트리거되어 요구사항 추출. Graph RAG 연동: RAG 엔진은 RFP 텍스트를 벡터 임베딩(예: OpenAI Embeddings)으로 변환 후 그래프 DB(Neo4j)에 저장. 요구사항 간 관계(예: 의존성, 우선순위)를 자동 매핑. 추가 파일 지원 (RFP 미존재 시): RFP 없이 요구사항 직접 입력 시, 보조 파일(이미지, 엑셀 등) 업로드를 허용. 이 파일도 Graph RAG로 분석되어 요구사항에 첨부. 요구사항 수정 기능 제공. 요구사항 수정시 그래프 RAG도 업데이트. 예: 요구사항 "UI 디자인 개선"에 디자인 스케치 파일 업로드 → 자동 태깅 및 그래프 노드 추가. 자동 ID 부여: 추출된 각 요구사항에 고유 ID(예: REQ-프로젝트ID-순번) 부여. ID는 UUID 기반으로 중복 방지.  2.2 요구사항과 스프린트 매핑  매핑 기능: 스프린트 생성 시 UI에서 요구사항 목록 표시, 드래그-앤-드롭 또는 체크박스로 매핑. 매핑 시 그래프 DB에 엣지 추가(요구사항 → 스프린트). 진행률 업데이트: 스프린트 진행 중 태스크 완료 시 요구사항 진행률 자동 계산(예: 매핑된 태스크 5개 중 3개 완료 → 60%). 상태 표시: "진행 중"(0-99%), "완료"(100%), "지연"(기한 초과 시 자동 플래그). 실시간 동기화: 웹훅 또는 이벤트 드리븐 아키텍처로 스프린트 업데이트 시 Graph RAG 재쿼리하여 대시보드 갱신.  2.3 테넌트 관리  프로젝트별 테넌트: 각 프로젝트를 별도 테넌트로 isolation. PMS 데이터베이스(MongoDB/PostgreSQL)에서 테넌트 ID로 파티셔닝. Graph RAG DB 테넌트 지원: Neo4j에서 Namespace 또는 Separate Graph로 테넌트 구분. 쿼리 시 테넌트 필터 적용(예: MATCH (n:Requirement {tenant: 'proj1'})). 장점: 데이터 보안 강화, 멀티 프로젝트 동시 운영, 스케일링 용이.  2.4 AI 보고서 생성 기능  담당자별 주간 보고: AI가 Graph RAG에서 담당자 할당 요구사항/스프린트 데이터를 쿼리하여 주간 요약 생성. 템플릿 기반(예: "이번 주 완료: X개, 진행 중: Y개, 이슈: Z").  사용자가 AI 보고서 생성 버튼 클릭시 생성, 이메일/PMS 알림 푸시. AI 모델(GPT-4o 등)로 자연어 보고서 작성. 커스터마이징: 사용자 설정으로 보고서 포맷(텍스트/PDF) 선택, 추가 메트릭(시간 추정 vs. 실제) 포함.  API 엔드포인트: /upload/rfp, /map/sprint, /report/generate. RAG 파이프라인: 문서 로더 → 스플리터 → 임베딩 → 그래프 스토어. Graph DB (Neo4j for RAG). 통합 레이어: Kafka 또는 Redis로 이벤트 큐잉, 실시간 업데이트 보장. 추가 파일 처리: S3 버킷 저장, Graph RAG에 메타데이터 인덱싱.  3.2 데이터 흐름  RFP/추가 파일 업로드 → 백엔드 API 호출 → 파일 저장 → Graph RAG 트리거 → 요구사항 추출/ID 부여 → 그래프 DB 저장. 스프린트 생성 → 매핑 API → 그래프 엣지 업데이트. 태스크 업데이트 → 진행률 계산 → 대시보드 푸시. 사용자 AI 주간 보고서 생성 → Graph RAG 쿼리 → AI 보고서 생성 → 사용자 알림.
